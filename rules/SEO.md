# ✅ Полный чек-лист SEO-оптимизации сайта

---

## 1. Техническое SEO (Crawl / Index / Дубли)

### 1.1. Доступность сайта
- [ ] Сайт открывается без ошибок  
- [ ] Нет 5xx / 4xx на рабочих страницах  
- [ ] Сервер стабильно отвечает  

---

### 1.2. Зеркала сайта
- [ ] Выбран один основной вариант:
  - https
  - www или без www  
- [ ] Все остальные варианты → 301  
- [ ] Нет цепочек редиректов  

---

### 1.3. HTTP-статусы
- [ ] Рабочие страницы → 200  
- [ ] Удалённые → 404 или 410  
- [ ] Перенесённые → 301  
- [ ] Нет 200 у несуществующих страниц (soft 404)  

---

### 1.4. Canonical
- [ ] Canonical есть на всех страницах  
- [ ] Canonical = текущий URL страницы  
- [ ] Canonical не указывает на редирект  
- [ ] Нет конфликтов canonical ↔ редирект  

---

### 1.5. XML Sitemap
- [ ] Sitemap доступен по одному URL  
- [ ] В sitemap:
  - [ ] только SEO-страницы  
  - [ ] нет мусора (users, служебные CPT)  
- [ ] Нет страниц с `noindex` в sitemap  
- [ ] Sitemap добавлен в поисковые системы  

---

### 1.6. robots.txt
- [ ] robots.txt доступен  
- [ ] Нет `Disallow: /` для `User-agent *`  
- [ ] Закрыт `/wp-admin/`  
- [ ] Разрешён `admin-ajax.php`  
- [ ] Sitemap указан (опционально)  

---

### 1.7. Дубли URL
- [ ] Нет дублей из-за:
  - [ ] слэшей  
  - [ ] параметров  
  - [ ] `page/1`  
- [ ] Все дубли решаются:
  - [ ] canonical  
  - [ ] либо редиректом (не вместе)  

---

### 1.8. Параметры URL
- [ ] Параметры не создают индексируемые дубли  
- [ ] Нет параметров в sitemap  
- [ ] Нет индексации мусорных параметров  

---

### 1.9. Скорость и рендеринг
- [ ] HTML отдаётся быстро  
- [ ] Нет критического CLS  
- [ ] Нет блокирующего JS для контента  
- [ ] Страницы рендерятся без ошибок  

---

### 1.10. Микроразметка (базовая)
- [ ] Schema.org:
  - Organization / MedicalBusiness / Dentist  
- [ ] JSON-LD  
- [ ] Нет ошибок в валидаторе  

---

## 2. SEO внутри `<html> … </html>` (On-Page)

### 2.1. `<title>`
- [ ] Есть на каждой странице  
- [ ] Уникальный  
- [ ] ≤ 60–65 символов  
- [ ] Не дублируется между страницами  
- [ ] Не совпадает 1 в 1 с H1  

---

### 2.2. `<meta description>`
- [ ] Есть на каждой странице  
- [ ] Уникальный  
- [ ] ≤ 150–160 символов  
- [ ] Не автоспам  
- [ ] Не пустой  

---

### 2.3. `<meta robots>`
- [ ] `index, follow` — для SEO-страниц  
- [ ] `noindex` — для служебных / мусорных  
- [ ] Нет конфликтов (sitemap + noindex ❌)  

---

### 2.4. H1
- [ ] Ровно один H1 на страницу  
- [ ] H1 отражает суть страницы  
- [ ] H1 не размножается в сквозных блоках  
- [ ] Нет пустого H1  

---

### 2.5. H2–H6
- [ ] Логичная иерархия  
- [ ] Нет пропусков уровней (H2 → H4 ❌)  
- [ ] Не используются для дизайна  
- [ ] Нет одинаковых SEO-заголовков на всех страницах  

---

### 2.6. HTML-структура
- [ ] Один `<main>`  
- [ ] `<header>` и `<footer>` используются корректно  
- [ ] Нет скрытого SEO-контента (`display:none`)  
- [ ] Нет контента, загружаемого только через JS  

---

### 2.7. Внутренние ссылки
- [ ] Все ссылки кликабельные (`<a href>`)  
- [ ] Нет `href="#"` в SEO-блоках  
- [ ] Нет битых внутренних ссылок  
- [ ] Анкоры читаемые (не «Подробнее»)  

---

### 2.8. Изображения
- [ ] `alt` у всех смысловых изображений  
- [ ] Нет SEO-спама в `alt`  
- [ ] Изображения не загружаются только через JS  

---

### 2.9. Сквозные блоки
- [ ] Нет одинаковых SEO H1/H2 на всех страницах  
- [ ] Нет повторяющихся SEO-текстов  
- [ ] Сквозные блоки нейтральные  

---

### 2.10. Локальное SEO (on-page)
- [ ] Адрес указан текстом  
- [ ] Телефон указан текстом  
- [ ] Название клиники единообразно  
- [ ] Контакты доступны с любой страницы  

---

## 3. Контроль и валидация
- [ ] Страницы индексируются  
- [ ] Нет ошибок в вебмастерах  
- [ ] Нет резких скачков ошибок  
- [ ] Sitemap принят  
- [ ] Canonical учитывается
